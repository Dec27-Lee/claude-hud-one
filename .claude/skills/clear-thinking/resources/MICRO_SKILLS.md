# clear-thinking 微技能索引

本文件只做索引，不存放完整微技能内容。运行时先由 `ROUTER.md` 判断问题处境，再只读取命中的分类文件，避免一次性拿到整个微技能库。

## 使用规则

1. 默认不要读取所有分类文件。
2. 先读 `ROUTER.md` 判断处境和候选微技能。
3. 只读取会改变当前判断的 1-2 个分类文件。
4. 如果跨多个处境，优先读取通用判断，再读取一个专项分类。
5. 如果需要 3 个以上分类文件，说明任务已偏大，应考虑子代理或 Workflow。

## 分类入口

| 问题处境 | 读取文件 | 常用微技能 |
| --- | --- | --- |
| 目标、事实、结构、取舍、执行、验证、完成检查 | `micro-skills/common.md` | 目标澄清、事实/推断/假设分离、问题分型、主要矛盾、最小行动、风险扫描、信息缺口、方案取舍、完成检查 |
| 产品、PRD、原型、页面、研发协作、字段/状态/权限 | `micro-skills/product-rd.md` | 需求四问、复用/扩展/新建判断、信息一二三档、页面范式、字段/状态/操作卡、质量裁决 |
| AI 工作台、Agent、组织产品、办公协同、默认值、责任状态 | `micro-skills/ai-org-product.md` | 发心拆层、定位四问、不可能三角、组件权力、责任状态、默认值制度、共创偏差、敏捷债务、组织叙事校准、身体边界与学徒性 |
| 创业、经营、增长、效率、口碑、长期产品、生态协同 | `micro-skills/business-long-term.md` | 顺势判断、用户朋友、爆品聚焦、效率革命、口碑飞轮、长期复利、生态协同 |
| 高反思、高归因、解释很多但难行动、身体/关系/责任边界 | `micro-skills/high-context.md` | 痛感分流、解释关闭、切断力 |
| 建设或更新思考方法本身、从材料中抽象判断动作 | `micro-skills/method-absorption.md` | 判断动作提取、微技能封装、反权威背书 |

## 组合建议

| 场景 | 最小读取组合 |
| --- | --- |
| 普通复杂判断 | `common.md` |
| 产品问题 | `common.md` + `product-rd.md` |
| AI / 组织产品问题 | `common.md` + `ai-org-product.md` |
| 创业经营或长期方向 | `common.md` + `business-long-term.md` |
| 高上下文内耗 | `common.md` + `high-context.md` |
| 技能建设或方法吸收 | `common.md` + `method-absorption.md` |

## 微技能统一格式

每个分类文件中的微技能都保持同一结构：

```markdown
- 适用处境：什么问题形态下调用
- 输入：需要哪些事实或材料
- 操作：如何思考
- 产出：形成什么中间结论
- 失效信号：何时停止或切换
```
