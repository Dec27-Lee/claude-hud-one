# 产品研发微技能

本文件只在 `ROUTER.md` 判断问题属于产品、PRD、原型、页面、研发协作、字段/状态/权限、验收时读取。

## 1. 需求四问

- 适用处境：需求被写成功能、按钮、页面、颜色、入口或“灵活一点”。
- 输入：需求原话、角色、场景。
- 操作：回答什么人、什么情况下、碰到什么问题、想达到什么目的。
- 产出：角色场景问题目标。
- 失效信号：四问已明确，继续追问会拖慢。

## 2. 复用 / 扩展 / 新建判断

- 适用处境：要新增功能或页面，但不确定是否已有能力可复用。
- 输入：现有模块、需求差异、未来扩展可能。
- 操作：判断复用、扩展、新建；标注配置项。
- 产出：建设方式和理由。
- 失效信号：完全是新领域且无复用对象。

## 3. 信息一二三档

- 适用处境：页面信息多、重点不清、首屏混乱。
- 输入：用户任务、字段、操作、角色。
- 操作：拆一档必看、二档辅助、三档低频；安排显隐和层级。
- 产出：信息架构。
- 失效信号：不是页面/文档/信息呈现问题。

## 4. 页面范式选择

- 适用处境：列表、详情、表单、看板、树表、统计明细等范式不确定。
- 输入：数据特征、操作模式、主从关系、扩展性。
- 操作：匹配成熟范式；找内部历史页面；说明依据。
- 产出：页面范式和选择理由。
- 失效信号：只是业务规则判断，不涉及界面结构。

## 5. 字段 / 状态 / 操作卡

- 适用处境：PRD、原型或代码需要明确字段、状态、权限和交互。
- 输入：业务规则、数据依据、角色权限、操作结果。
- 操作：为字段、状态、操作分别写依据、规则、可见性、反馈和异常。
- 产出：可研发实现和测试的对象卡。
- 失效信号：处于早期方向判断，还不该细化字段。

## 6. 质量裁决

- 适用处境：交付前检查产品/PRD/原型/代码协作质量。
- 输入：需求、架构、范式、字段、权限、交互、验收。
- 操作：按 PASS / WARN / FAIL 裁决。
- 产出：阻塞项、警告项、通过项。
- 失效信号：还没有具体方案可检查。
