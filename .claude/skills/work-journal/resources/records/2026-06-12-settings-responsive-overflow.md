# 设置页终端 HUD 设置体验优化

## 原始需求

第一轮：用户反馈设置页在非全屏窗口下存在组件溢出：

- 终端 HUD > 组件 > 组件参数区域部分控件被右侧裁切。
- 终端 HUD > 颜色配置右侧部分组件超出容器，全屏后问题消失。
- 需要检查各组件样式，判断并修复自适应问题。

第二轮：用户继续指出终端 HUD 设置页面需要优化：

1. 预览区域、组件/颜色/JSON tab 栏固定在顶部；组件库、行配置、组件参数是三个独立卡片，并根据配置界面大小各自自适应上下滚动。
2. 行添加组件方式不能只有选择，需要支持从组件库拖拽过去。
3. 组件库组件名称在简体中文设置页下主文字显示中文、次文字显示英文名。
4. 组件参数配置在简体中文设置页下显示中文，英文模式显示英文。
5. 最大宽度设置输入框高度要和左侧 tab 切换一致；最大宽度和输入框左右结构并对齐。
6. 移除诊断页面。

第三轮：用户反馈“长按组件库的组件还是无法拖拽”，需要修复真实界面里组件库拖拽不可用的问题。

第四轮：用户继续要求：

1. 不需要“选择添加”操作，删除行配置里的选择添加下拉。
2. 已经在行配置里的组件，在组件库里不再显示。

## 范围

- 本轮做：终端 HUD 设置页顶部固定、三卡片独立滚动、拖拽添加组件、组件库/参数本地化、最大宽度布局对齐、移除诊断 tab/page、修复真实界面长按/拖动组件库不可用问题、删除选择添加下拉、组件库过滤已配置组件、更新测试、构建与安装包打包。
- 本轮不做：重新设计整套 Settings 视觉、修改 Terminal HUD 运行时渲染模型、提交/推送。
- 待确认：如需更精确视觉验收，后续可用真实 Tauri 窗口截图继续微调。

## 计划

1. 定位 `TerminalHudPanel.tsx` 和 `styles.css` 当前结构。
2. 调整组件工作区布局：顶部固定，三列卡片内部独立滚动并保持响应式。
3. 完善拖拽添加：从组件库拖入行 dropzone 即可添加。
4. 增加中英文展示映射：组件库和组件参数在中文/英文语言下切换标签。
5. 调整最大宽度控件结构与高度；移除诊断 tab/page。
6. 更新 Playwright 测试，运行 build、UI 测试和 `npm run tauri:build`。
